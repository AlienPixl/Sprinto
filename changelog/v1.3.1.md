## Highlights

- Room queue layout now scales more naturally with taller desktop windows, showing more visible issues before pagination while keeping the footer visible.
- Room resizing behavior is now more stable in both directions, so shrinking the browser height reduces the visible queue capacity instead of leaving the old taller layout behind.

## Bug Fixes

- Fixed queue filter/sort controls in the room view so they use cleaner icons, stay side by side on desktop, and keep dropdown text readable in Microsoft Edge on Windows.
- Fixed room footer and queue-list interactions on short desktop windows by keeping the footer anchored and moving overflow into the room content area instead of letting the queue panel spill past the page bottom.
- Fixed mobile room regressions introduced by the desktop layout work so the stacked room controls, queue form, and issue list render in normal document flow again on phone-width screens.
