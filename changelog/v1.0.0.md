## Highlights

- Sprinto 1.0.0 packages planning poker rooms, administration, authentication, and Jira integrations as the first stable product release.
- Jira worklog reporting, Jira room actions, and product administration are now available together in one versioned release line.
- Docker-based installation and versioned GitHub releases now define the supported release path for Sprinto.

## Included In 1.0.0

- Planning poker rooms with realtime voting, reveal flow, issue queues, history playback, and role-based room actions.
- Administration for users, roles, decks, settings, scheduled tasks, audit logs, active sessions, and update checks.
- Authentication options for local accounts, Microsoft Active Directory, Microsoft Entra, and a deployment-managed recovery admin.
- Jira integrations for room-side issue import, Jira delivery actions, Jira PDF reporting, and compact Jira Worklog reports with grouped tables and charts.
- Docker-first deployment with bundled PostgreSQL for local startup and support for external PostgreSQL deployments.

## Release Notes

- Release assets include a Linux `amd64` Docker image tarball and a release-ready Docker Compose file.
- Update checks are now expected to resolve against `AlienPixl/Sprinto`.
