## Features

- Admins can now configure a **report timezone** in Settings → Date and Time. Jira comments and PDF reports format all timestamps in the configured zone and include the UTC offset (e.g. `10. 6. 2026 10:45 UTC+2`). The dropdown lists ~30 common IANA timezones grouped by region, each showing the current UTC offset. Default is `UTC`.

## Bug Fixes

- Scrubbing through a completed issue's timeline now updates votes and participants exactly when the scrubber crosses each event dot. Previously, all votes could appear at once when several participants voted within the same short window, because state changes were driven by raw timestamps rather than by the scrubber's visual position on the timeline. The card reveal now also triggers at the exact position of the Reveal anchor.
