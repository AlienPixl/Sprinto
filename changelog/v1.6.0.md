## Features

- Admins can now configure a **default card highlight mode** (No highlight / Most frequented card / Highest value card) in the Rooms admin settings. New rooms are created with the configured mode applied. The Room settings panel in the room itself uses the same option labels. The Rooms admin section has been restructured: Default timer, Default deck, Default card highlight, and Require story ID for manual issues are grouped under a **General** collapsible

- Admins can now configure **default Jira import filter conditions** in the Jira Cloud integration settings. The editor supports all three field types (Story Points, Original Estimate, Status) with AND/OR connectors, matching the filter editor available in the room import panel. When a room import panel is first opened, the filter is pre-filled with the admin-configured default. Users can change the filters for their session without affecting the global default.

- When a room is **closed**, the current issue title is replaced with a dash (—) in both the room view and the dashboard card. The issue-banner row itself remains visible so the layout stays stable; the Jira link button is also hidden while the room is closed. History playback inside a closed room is unaffected and still shows the real issue title.

- When a Jira issue is started from the queue, the linked Jira issue **automatically opens in a new tab** for all participants. The behaviour can be turned off per room via a toggle in the Room settings panel (*Open Jira issue on start*). It is enabled by default.

- The standalone **Highlight** button in the room controls panel has been replaced by a **Room settings** button. Clicking it opens a settings overlay where card highlight mode can be changed via a labelled dropdown. The panel is designed to accommodate additional per-room settings over time.

- Dashboard room cards now show a **voter and viewer breakdown** alongside the total participant count — for example *3 participants (2 voters, 1 viewers)*. Voters are participants who hold the vote permission; viewers are all other present participants.

- Failed login attempts are now recorded in the audit log. Covers wrong credentials (local and Active Directory), AD users who have no Sprinto role group assigned, AD accounts that have already been migrated to Microsoft Entra, and Microsoft Entra sign-in failures (provider error, cancelled flow, state mismatch). All events are stored without attaching the unverified identifier to a Sprinto user account.

- **Queue sort is now a shared room setting.** Changing the sort order (Issue key / Reporter / Priority) in the queue panel updates all participants in the room simultaneously. The selected sort is persisted and restored when participants reconnect.

- Queue issue rows now show Reporter, Priority, and State in a right-aligned meta strip before the **JIRA** / **MAN** source tag and link icon. Priority is fixed-width and centered; vertical separators appear between each meta field and before the source tag.

- Room moderators can now **return the current issue to the queue** without having to reveal cards first. A **Return to queue** button appears in the controls panel while voting is in progress. Clicking it opens a confirmation dialog; on confirm, the active issue moves back to the queue, all votes cast in that round are discarded, and the room returns to open status. This replaces the previous situation where the only way to move on from an issue was to reveal it, which marked it as completed even with no votes.

- Rename room dialog now includes a category picker when room categories are enabled. The category is pre-filled with the room's current value and behaves as optional or required based on the admin-configured setting, matching the create-room form behaviour.

- Room admin settings now include a dedicated **Issue list** section containing a **Default issue sort** selector (Issue key / Reporter / Priority) and the **Require story ID** toggle. The chosen sort order is applied automatically when any room is opened; users can still change it for their current session.

- The **Import order** field has been removed from the Jira room import dialog; the room queue sort control already covers this. Issues are always imported in issue-key order.

- Jira import filter conditions now support a **Status** field in addition to Story Points and Original Estimate. Available operators are **IN** and **NOT IN**; statuses are loaded live from Jira when the import panel opens. The value picker is a custom dropdown with a button-per-status list and a checkmark indicator for selected items.

## Bug fixes

- Jira import filter condition **× remove** button and **+ Add condition** button were invisible in Sprinto Glow due to light-on-light colour values.

- Jira import filter **AND / OR** connector was unreadably styled in Sprinto Glow.

- Jira import filter **AND / OR** connector was incorrectly tall (inherited the global 48 px select min-height) and stretched to the full row width instead of sizing to its content.

- Jira import filter condition operator dropdowns were unreadable on Windows in both themes.

- Jira assignee picker now shows only participants who can vote in the room-participants section of the dropdown.

- Room issue queue now correctly fills the available window height and adjusts the number of visible items per page when the browser window is resized or maximized.

- Navigating to a later queue page and then resizing the window to a larger size no longer leaves an empty issue list; the current page is automatically reset to the last valid page.

- README animated screencasts now display on GitHub; the video embeds were replaced with GIFs and the dynamic license badge was fixed by including the version file in the public release export.
