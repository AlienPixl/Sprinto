## Highlights

- Jira Worklog now supports much richer Jira search and filtering, while larger report ranges load significantly faster.
- Public release packages now ship safer Docker deployment templates that must be reviewed before first startup.

## Changes

- Jira Worklog: the issue picker now supports both Jira issues and matching Jira projects/spaces in one wider `ID / Name / Type` result list.
- Jira Worklog: issue types in worklog search are normalized to English labels such as `Task`, `Story`, `Epic`, and `Project`.
- Jira Worklog: report rows now link issue keys directly to the matching Jira issue in a new tab.
- Jira Worklog: the `User or group` picker now supports Jira groups in addition to Jira users.
- Jira Worklog: primary and secondary grouping now support the richer worklog dimensions, and the toolbar keeps `Epic - Include children` available as its own explicit toggle.
- Jira Worklog: report generation now narrows candidate issues in Jira first by worklog date and fetches only time-bounded worklogs, which makes wider report ranges noticeably faster.
- Public release packaging: releases now include `.env.example` as its own asset and export the Docker deployment file as `docker-compose.example.yml` instead of a ready-to-run `docker-compose.yml`.
