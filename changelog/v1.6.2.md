## Bug Fixes

- Fixed *Open Jira issue on start* opening a Jira tab in **every** participant's browser instead of only for the person who started the issue. The tab-open logic was tied to a WebSocket snapshot update that fires on all connected clients simultaneously. It now fires exclusively on the client that clicked the start button.

- New rooms now inherit the admin-configured default issue sort order instead of always starting with *Issue key*. Previously the `Default issue sort` admin setting had no effect on newly created rooms — the database column default (`'issue'`) was always used regardless. The room creation endpoint now reads the configured default and passes it to the insert, so a room created after setting the default to *Priority* or *Reporter* immediately opens with that sort order. Existing rooms retain their current per-room sort setting unaffected.
