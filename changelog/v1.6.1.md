## Bug Fixes

- Jira import status filter now matches statuses by their Jira ID instead of their display name. Previously, two statuses with identical translated names (e.g. both named *Zrušené* in Czech) were indistinguishable, causing both to be selected when only one was chosen. Switching to the stable numeric Jira status ID removes the ambiguity regardless of locale.

- Dashboard participant and voter counts now update immediately when a participant closes their tab or loses their connection. Previously, presence was cleared only when the browser managed to send an explicit leave request, so crashed tabs or dropped connections would leave the participant counted until the next page reload. The WebSocket connection is now used as the presence signal — the moment it closes, the participant is removed and the dashboard reflects the real current state.
