## Highlights

- Login screen spacing is now balanced in both the local-only and Microsoft-enabled sign-in layouts.
- Authentication settings now make save outcomes clearer with visible required fields and reliable save feedback.

## Changes

- Frontend/authentication: Removed the fixed minimum content height from the login card so it no longer leaves unnecessary empty space below the visible sign-in actions.
- Admin/authentication: `Save settings` now reliably flips to `✓ Saved` after successful auth changes, shows inline validation errors when save is rejected, and marks save-blocking fields with `(required)` when their provider or transport mode is enabled.

- `npm run test:run -- AdminPanel ad.test.ts`
- `npm run build`
