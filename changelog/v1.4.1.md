## Bug fixes

- Timeline density / clustering in history and PDF exports

  - Events that happened in a short time window (e.g., many users voting nearly simultaneously) could previously stack on a single point at the start or end of the timeline. Timeline markers in both the web history and the exported PDF are now laid out evenly between `Start` and `Reveal` so dense rounds remain readable.
  - Files: `app/web/src/components/RoomView.tsx`, `app/src/jira.js`, `docs/ai-contexts.md`

## Notes

- This change preserves chronological ordering of events while improving visual distribution for readability. If a different behavior is desired (preserve timestamp positioning unless a dense cluster is detected), we can add a toggle or heuristic in a follow-up.
